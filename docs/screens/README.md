# Reference screenshots

Generated by `bun run e2e:screenshots` (Playwright) against the Vite dev
server. The shell renders without a Tauri backend, so the screenshots
land in their pre-data empty states — that's intentional for the
reference set; production data captures need a packaged binary E2E
pass, which is deferred.

Re-generate after any change that touches layout, brand tokens, or the
empty-state copy. Commit the resulting PNGs alongside the change so
the next reviewer can diff visually.
